# Phase 5.5 — UI Refinement Plan
## GENESIS App · Expo SDK 54 + NativeWind v4
**Date:** 2026-02-08
**Author:** Claude (brainstorm with Aldo)
**Status:** Ready for Implementation

## Executive Summary

Phase 5.5 bridges the gap between the current prototype (Phases 1–5) and Phase 6 (Functional MVP). The app has 20 screens, 53 components, and ~51K lines of code. The structure is solid. The visual language needs to match the premium reference designs Aldo has been developing.

**Core insight:** The 11 HTML reference mockups define a clear visual DNA that the current React Native app only partially implements. This plan extracts the gold from those references and maps it to each of the 7 refinement areas.

### B1. HOME — Genesis Briefing Card

**File:** `app/(tabs)/home.tsx`
**Current State:** Flat ImageCard with text overlay. Phase label uses `phaseConfig.accentColor`. No glassmorphism, no glow, no depth.

**What the References Show:**
- Glass card with `backdrop-blur(20px)` and `rgba(10, 10, 10, 0.85)` background
- Subtle white border at 8% opacity
- `#6D00FF` glow shadow on the card
- Staggered entry animations (cards fade in sequentially)
- Chip-style badges ("HYBRID", "SEASON 1") with pill shape and purple accent
- Hero metrics row (kcal, sleep, water, steps) in a grid below the main card
- Chrome metallic accent on secondary text

**Proposal — "Liquid Glass Briefing":**

1. **Replace ImageCard with GlassCard** for the Genesis Briefing section:
   - Background: `rgba(10, 10, 10, 0.85)` with 1px border `rgba(255,255,255,0.08)`
   - Border-radius: 20px (currently using rounded-2xl which is 16px — bump to 20)
   - Add glow shadow: `shadowColor: '#6D00FF', shadowOpacity: 0.25, shadowRadius: 15`

2. **Add phase badge as chip:**
   - Pill shape (`borderRadius: 9999`)
   - Background: `rgba(109, 0, 255, 0.1)`
   - Border: `1px solid rgba(109, 0, 255, 0.4)`
   - Text: `#6D00FF`, uppercase, 10px, JetBrains Mono, tracking-widest

3. **Briefing text treatment:**
   - Mission title in Inter Bold, 18px, white
   - Body text in Inter, 14px, `rgba(255, 255, 255, 0.60)`
   - Chrome-tinted accent line for metadata: `rgba(192, 192, 192, 0.60)`

4. **Add 4-metric row below briefing card:**
   - Grid of 4 small glass cards (kcal, sleep, water, steps)
   - Each: icon + value in JetBrains Mono + label in Chrome text
   - Use semantic colors: Coral for kcal, Ocean for sleep, Cyan for water, Gold for steps

5. **Staggered entry animation:**
   - Each card enters with `opacity: 0 → 1`, `translateY: 10 → 0`
   - Delay: 80ms per card (0, 80, 160, 240ms)
   - Use `Animated.timing` with `useNativeDriver: true`

**Files to modify:**
- `app/(tabs)/home.tsx` — restructure briefing section
- `components/ui/GlassCard.tsx` — enhance with glow variant prop
- `constants/Colors.ts` — new color values

### B6. FUEL PAGE — Scroll Fix + Visual Alignment

**File:** `app/(tabs)/fuel.tsx`
**Current State:** ScrollView with paddingBottom: 100. Same structure as Home (which works). Content reportedly cut off.

**Diagnosis:** The ScrollView structure is identical to Home. The scroll issue is likely caused by:
1. Content height + paddingBottom not accounting for CustomTabBar height
2. SafeAreaView bottom inset not propagating correctly
3. Possible conflict between gesture handler ScrollView and SafeAreaView

**Proposal — Scroll Fix + Visual Refresh:**

1. **Scroll fix (try in order):**
   ```typescript
   // Option A: Increase bottom padding to clear tab bar
   contentContainerStyle={{ paddingBottom: 120 }}

   // Option B: Use useBottomTabBarHeight() from @react-navigation
   import { useBottomTabBarHeight } from '@react-navigation/bottom-tabs';
   const tabBarHeight = useBottomTabBarHeight();
   contentContainerStyle={{ paddingBottom: tabBarHeight + 20 }}

   // Option C: If SafeAreaView is the issue, use edges prop
   <SafeAreaView edges={['top']} style={{ flex: 1 }}>
     <ScrollView contentContainerStyle={{ paddingBottom: 120 }}>
   ```

2. **Visual refresh (to match references):**
   - Calorie CircularProgress: Replace current accent color with brand violet + glow
   - MacroCards: Use glass card styling with subtle borders
   - Meal ImageCards: Add gradient overlay fade
   - WaterDots: Use `#00E5FF` (cyan) instead of current color
   - Section headers: JetBrains Mono, 10px, uppercase, chrome text

3. **Color fixes:** Replace all `#b39aff` references with `GENESIS_COLORS.primary`

**Files to modify:**
- `app/(tabs)/fuel.tsx` — scroll fix + color updates

## SECTION F — CONSTRAINTS & NOTES

1. **No new dependencies** — Use existing: expo-linear-gradient, react-native-gesture-handler, lucide-react-native, NativeWind
2. **No Animated API for now** — Staggered animations in B1 are nice-to-have. If time is short, skip animations and focus on color + layout.
3. **Ambient orbs are web-only** — The reference HTML uses CSS blur orbs. In React Native, this would require reanimated + blur views. Defer to Phase 6.
4. **Scanlines/noise overlays are web-only** — Skip for mobile.
5. **Chrome metallic gradient** — Can be approximated with `LinearGradient` but is optional. Solid chrome colors are sufficient.
6. **Test on iOS and Android** — Blur effects (backdrop-filter) may not work on Android. Use solid semi-transparent backgrounds as fallback.
